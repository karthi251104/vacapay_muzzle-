# Complete Workflow And Evaluation

This document explains the current cattle muzzle field-testing workflow from capture to backend embedding, Pinecone search, and admin evaluation.

## 1. Two Workflows

The app should use only these workflow names:

```text
Cattle Enrolment
Cattle Search
```

Cattle enrolment creates a permanent cattle identity.

Cattle search tests a captured cattle against enrolled identities. A cattle search may return an existing cattle, or it may correctly return no cattle found.

Use cattle search in UI and reports.

## 2. Farmer Flow

Agent starts from the first screen:

```text
Add Farmer
Search Farmer
```

Add Farmer:

```text
farmer name
GPS location
unique farmer ID generated by app/backend
start first cattle enrolment under farmer
```

Farmer IDs are unique and non-sequential:

```text
FARM-A7K9Q2M4
FARM-P4T8X6JD
FARM-W29LQ7RA
```

The frontend shows a clean generated ID. The backend makes the final ID decision using saved records so multiple agents do not collide.

Search Farmer:

```text
GPS search shows farmers near the current location
name/ID search shows matching farmers
agent selects farmer
agent starts cattle enrolment or cattle search under that farmer
```

## 3. Offline Android Capture

The production Android app should do detection and quality checking on-device.

```text
camera frame
-> YOLO detects muzzle box offline
-> good/bad classifier checks image quality offline
-> bad image is rejected immediately
-> good image is cropped on phone
-> CLAHE is applied on phone
-> cropped image is saved locally
```

The backend should not be called for live YOLO detection in low-internet field areas.

Recommended Android model locations:

```text
android/app/src/main/assets/models/muzzle_detector.tflite
android/app/src/main/assets/models/muzzle_quality_classifier.tflite
```

The existing backend `best_v4.pt` is a PyTorch YOLO file for server/browser fallback. Android needs exported mobile models such as TFLite.

## 4. Muzzle Count

For faster field testing:

```text
3 good muzzle images
```

Files:

```text
muzzle1.jpg
muzzle2.jpg
muzzle3.jpg
```

Later this can be moved back to 5 by changing the shared muzzle count.

## 5. Upload Flow

Field officer can capture offline during field work.

Upload should happen later from a separate upload screen when internet is good.

```text
local cattle enrolment/search record
-> 3 cropped CLAHE muzzle images
-> 7 supporting images
-> upload API
-> backend saves images
-> backend creates DINOv2 embedding
-> backend writes database/Pinecone
```

For Android-preprocessed crops, upload with:

```text
clientProcessed=true
```

Then backend saves the crop as-is and does not run backend YOLO cropping.

The web tester can still use backend YOLO fallback while the native Android offline flow is being built.

## 6. Embedding Creation

For each uploaded muzzle crop:

```text
muzzle1 -> DINOv2 embedding 1
muzzle2 -> DINOv2 embedding 2
muzzle3 -> DINOv2 embedding 3
```

Backend averages the 3 embeddings into one capture embedding:

```text
average embedding = mean(embedding1, embedding2, embedding3)
```

This average embedding is used for cattle enrolment storage or cattle search comparison.

## 7. Pinecone Separation

Use separate Pinecone namespaces:

```text
vacapay-cattle-enrolment
vacapay-cattle-search
```

Cattle enrolment namespace:

```text
permanent cattle identities
used as search gallery
```

Cattle search namespace:

```text
field test/search evidence
not used as the main identity gallery
```

Search compares the cattle search embedding against the cattle enrolment namespace.

## 8. Search Order

Cattle search should be easy for field officers:

```text
1. selected farmer cattle
2. all enrolled cattle
```

Result source tags:

```text
farmer_cattle      -> match is under selected farmer
all_other_muzzle   -> match is from wider enrolled cattle database
```

This keeps farmer context useful while still catching cattle enrolled under another farmer.

## 9. Top-K Ranking

Top-k should rank cattle identities, not repeated files.

Old session wording was confusing. It meant two captures of the same cattle.

Use this explanation instead:

```text
Cattle A enrolment vector -> 91%
Cattle A old search evidence -> 88%
Cattle B enrolment vector -> 86%
```

Metric Top-k becomes:

```text
1. Cattle A -> 91%
2. Cattle B -> 86%
```

The same cattle should not occupy multiple Top-5 positions.

## 10. Result Types

Cattle search can produce:

```text
matched_existing
new_cattle / no_cattle_found
```

Matched existing means the app found an enrolled cattle above threshold.

No cattle found means no enrolled cattle passed threshold. This can be correct when the tested cattle is new.

## 11. Admin Ground Truth

Admin reviews supporting photos and sets the real answer.

For a matched search:

```text
expected cattle = matched cattle -> Correct Match
expected cattle != matched cattle -> Wrong Match
```

For a no-cattle-found search:

```text
expected cattle is an existing enrolled cattle -> Missed Match
captured cattle is actually new -> Correct No Cattle Found
```

## 12. Metrics

Admin dashboard should show:

```text
Registered Cattle
Cattle Searches
Reviewed Searches
Correct Matches
Correct No Cattle Found
Missed Matches
Wrong Matches
Top-1 Accuracy
Top-5 Accuracy
False Matches
Officer-wise capture quality
```

Top-1 Accuracy:

```text
reviewed searches where expected cattle is rank 1 / reviewed searches
```

Top-5 Accuracy:

```text
reviewed searches where expected cattle appears in Top-5 / reviewed searches
```

Correct No Cattle Found is counted separately because Top-k is not applicable when the expected result is no enrolled cattle.

## 13. Field Test Plan

For 10 field officers:

```text
50 cattle enrolments per officer
50 cattle searches per officer
```

Total target:

```text
500 cattle enrolments
500 cattle searches
```

The cattle search set should include:

```text
already enrolled cattle -> expected result is correct cattle ID
new cattle -> expected result is no cattle found
```

This gives a realistic field test because officers will see both known and unknown cattle.
