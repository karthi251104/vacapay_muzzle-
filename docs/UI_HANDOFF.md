# UI Teammate Handoff

This document is for the teammate improving the UI/design.

## 1. Where UI Code Lives

Main files:

```text
frontend/src/app/app.component.html
frontend/src/app/app.component.css
frontend/src/app/app.component.ts
frontend/src/app/api.service.ts
```

Most visual work is in:

```text
frontend/src/app/app.component.html

```

Use `app.component.ts` only when changing screen logic, button behavior, state, or labels generated by TypeScript getters.

## 2. App Screens

The app has two main roles.

### Admin screens

Admin can see:

- login
- agent creation
- agents list
- farm/cattle records
- image viewer
- selected cattle detail
- visits/sessions
- ZIP download
- duplicate merge tool
- match review panel

### Field agent screens

Agent flow screens:

1. Home
2. Owner Details
3. GPS / Nearby Cattle
4. Muzzle Photos
5. Other Photos
6. Check & Save

The active field-agent screen is controlled by:

```ts
agentScreen: AgentScreen
```

in:

```text
frontend/src/app/app.component.ts
```

## 3. Important UX Rules

This app is for field agents, not farmers.

Design should be:

- mobile-first
- clear in sunlight/outdoor use
- easy for non-technical users
- large tap targets
- simple words
- low confusion around cattle IDs
- owner ID and photo progress should be clear
- camera/muzzle capture should feel guided

Avoid showing raw UUIDs to field agents unless necessary. Admin can see UUIDs.

## 4. Current Field-Agent Flow Details

### Home

Purpose:

- show stats
- show recent cattle
- start new capture

Primary action:

```text
Start Enrollment
```

### Owner Details

Inputs:

- Owner ID: required
- Owner Name: optional but useful
- Field Officer: auto/user editable

Field agent does not enter cattle ID.

### GPS / Nearby

Actions:

- Use GPS
- Check Existing
- Start Capture

Backend may automatically reuse an existing cattle folder if there is exactly one clear owner/GPS match.

### Muzzle Photos

Purpose:

- camera view
- auto capture
- bounding box display
- 5 muzzle slots
- model status cards

Important visual states:

- camera off
- camera on
- auto capture active
- detecting
- saved muzzle thumbnails
- detection failed/retake message

### Other Photos

Upload/capture slots:

- face1
- face2
- face3
- leftside
- rightside
- back
- udder

### Check & Save

Shows:

- owner ID
- owner/farmer name
- image count
- folder
- match result
- gallery preview

## 5. Admin UI Notes

Admin list may show raw cattle IDs because admin needs exact record management.

Admin merge flow:

1. Click/select the correct main cattle row.
2. Tick duplicate cattle rows.
3. Click `Merge into selected`.

This is mainly for old duplicate data cleanup.

## 8. How UI Teammate Runs Locally

Recommended for UI work without Docker:

Terminal 1 backend:

```powershell
cd "F:\vacapay\vacapay muzzle\backend"
.\.venv\Scripts\Activate.ps1
$env:PYTHON_BIN=(Resolve-Path .\.venv\Scripts\python.exe).Path
node src/server.js
```

Terminal 2 frontend:

```powershell
cd "F:\vacapay\vacapay muzzle\frontend"
pnpm start
```

Open:

```text
http://localhost:4200
```

If backend is already running in Docker on port 3000, UI teammate can run only frontend on port 4200.
